## Роль — `roles/<name>/`

Роль — это рецепт. Один рецепт = одна задача (dns, nftables, nginx).
Роль не знает на каком сервере запускается и не содержит конкретных значений — только переменные.

Внутри роли три папки, каждая запускается в своё время:

```
defaults/main.yml   — переменные по умолчанию, загружаются до старта
                      перебиваются если vars заданы в плейбуке сервера

tasks/main.yml      — сами действия, выполняются сверху вниз
                      здесь вызываются templates когда нужно сгенерировать конфиг

handlers/main.yml   — запускается после всех tasks, только если было изменение
                      если tasks не поменяли ничего — handler не запустится вообще
                      если notify сработал 3 раза — handler всё равно запустится один раз
```

Порядок: `defaults` → `tasks` → `handlers`

Если внутри роли есть IP, имя сервера или конкретный порт — это ошибка, значение не там.

## Создать новую роль

Скопируй папку `role_template`, переименуй в название роли, меняй содержимое.
`templates/` можно не копировать если роль не генерирует конфиги.

Единственное жёсткое правило: имя в `notify:` и имя в `handlers/main.yml` должны совпадать дословно — иначе handler не сработает.

```
tasks/main.yml
  notify: Reload nginx      ←─┐ должны быть
                               │ одинаковыми
handlers/main.yml              │
  - name: Reload nginx      ←─┘
```

## Для всех серверов — `servers_main.yml`

- Всё что нужно **каждому** серверу без исключения — здесь.
- Конкретные значения общих переменных (dns серверы, версии) — здесь в `vars:`.
- Список ролей только те, что реально нужны всем.

## Для конкретного сервера — `servers/<hostname>.yml`

- Один файл = один сервер. Имя файла совпадает с именем хоста в inventory.
- Только то, что делает этот сервер уникальным — список ролей и их переменные.
- Если задача нужна двум серверам из трёх — это сигнал подумать, не переехать ли ей в `servers_main.yml`.

## Общее правило

Каждый раз когда добавляешь что-то новое — один вопрос:

> **Это нужно всем серверам или одному конкретному?**

Всем → `servers_main.yml`
Одному → `servers/<hostname>.yml`
Логика как это делается → `roles/`
