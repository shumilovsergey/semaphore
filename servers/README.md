## Серверы — `servers/`

Один файл = один сервер. Имя файла совпадает с именем хоста в inventory.

Файл сервера — это **данные**. В нём перечислены роли для этого хоста и их переменные:

```yaml
server_roles:
  - prometheus
  - node_exporter

# переменные ролей — перебивают defaults роли
# node_exporter_url: ""
```

### `run.yml` — главный плейбук

`run.yml` лежит в корне проекта. Запускается на выбранной цели, для каждого хоста читает его `servers/<hostname>.yml` и выполняет перечисленные в нём роли. Цель передаётся через `target` в Extra Variables в Semaphore:

- `target: all` — все серверы
- `target: <группа>` — группа из inventory
- `target: <hostname>` — конкретный хост

`preflight` запускается всегда и первым — в `server_roles` его указывать не нужно. Если у хоста нет своего файла, выполнится только `preflight`.

### `1_default.yml`

Отдельный плейбук общей настройки (`preflight`, `service_user`, `server_info`). Запускается на `target` без файлов серверов — удобно для первичной настройки нового хоста.

## Добавить новый сервер

Скопируй `0_server_template.yml`, переименуй в `<hostname>.yml`, впиши нужные роли в `server_roles` и заполни их переменные.

## Preflight

Первая роль для каждого сервера — `preflight`. `run.yml` добавляет её автоматически.

Она запускается до всего остального и выполняет набор проверок. Если проверки прошли — выполнение продолжается. Если что-то не так — скрипт останавливается с описанием проблемы до того, как будут внесены какие-либо изменения на сервере.

Что именно проверяет `preflight` — смотри в `roles/preflight/tasks/main.yml`.
