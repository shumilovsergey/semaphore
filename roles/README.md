## Роль — `roles/<name>/`

Роль — это рецепт. Один рецепт = одна задача (dns, nftables, nginx).
Роль не знает на каком сервере запускается и не содержит конкретных значений — только переменные.

Внутри роли пять папок, каждая запускается в своё время:

```
defaults/main.yml   — переменные по умолчанию, загружаются до старта
                      перебиваются если vars заданы в плейбуке сервера

tasks/main.yml      — сами действия, выполняются сверху вниз
                      здесь вызываются templates когда нужно сгенерировать конфиг

handlers/main.yml   — запускается после всех tasks, только если было изменение
                      если tasks не поменяли ничего — handler не запустится вообще
                      если notify сработал 3 раза — handler всё равно запустится один раз

files/              — статические файлы (копируются через copy:)

templates/          — jinja2 шаблоны (.j2), переменные подставляются через template:
```

Порядок: `defaults` → `tasks` → `handlers`

Если внутри роли есть IP, имя сервера или конкретный порт — это ошибка, значение не там.

## Создать новую роль

Скопируй папку `0_role_template`, переименуй в название роли, меняй содержимое.
`files/` и `templates/` можно не копировать если роль их не использует.

Единственное жёсткое правило: имя в `notify:` и имя в `handlers/main.yml` должны совпадать дословно — иначе handler не сработает.

```
tasks/main.yml
  notify: Reload nginx      ←─┐ должны быть
                               │ одинаковыми
handlers/main.yml              │
  - name: Reload nginx      ←─┘
```
